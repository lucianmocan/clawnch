## KeychainCredStore (macOS, MVP)

Implementation shells out to the `security` CLI:

```go
func (k *KeychainCredStore) Get(key string) (string, error) {
    cmd := exec.Command("security", "find-generic-password",
        "-s", "clawnch",
        "-a", key,
        "-w")
    out, err := cmd.Output()
    if err != nil {
        return "", ErrCredNotFound{Key: key}
    }
    return strings.TrimSpace(string(out)), nil
}
```

**Service name:** `clawnch`

**Access control:** Entries created with `-A` (allow all applications to access without prompting) to avoid repeated dialog boxes.

**Security properties:**
- Keychain data is encrypted at rest with the user's login password
- On Apple Silicon, keychain encryption uses the Secure Enclave
- Credentials are never written to disk as plaintext by clawnch

## EncFileCredStore (Cross-Platform Fallback, MVP)

Uses `openssl enc -aes-256-cbc -pbkdf2 -iter 600000` for encryption.

- Algorithm: AES-256-CBC
- KDF: PBKDF2 with 600,000 iterations
- File format: OpenSSL's salted format
- Password: prompted on first use, cached in memory

Useful for Linux systems without a Secret Service daemon and headless/CI environments.

## SecretServiceCredStore (Linux, Future)

Shells out to `secret-tool`:

```sh
secret-tool store --label='clawnch' service clawnch key <key>
secret-tool lookup service clawnch key <key>
```

## Credential Flow

1. Agent declares required env vars via `Agent.EnvKeys`
2. For each key, `credStore.Get(key)` is called
3. Retrieved values are written to a temp file (`mktemp`, `0600`)
4. The temp file is passed to Docker via `--env-file`
5. Temp file is deleted on exit via `defer os.Remove()` and signal trap

## Temp File Security

- Created with `mktemp` (non-deterministic name)
- Permission set to `0600` (owner read/write only)
- Registered with `defer os.Remove(file)` and signal trap (SIGINT, SIGTERM)
- Future improvement: use a pipe instead of a temp file to avoid disk writes entirely
